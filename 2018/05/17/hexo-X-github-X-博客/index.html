github.io

github page是由github提供的静态网站托管服务（免费且对没有非常多亿恒泰图片的静态网页来说几乎没有空间限制，当然超过1G会可能会收到Linus的律师函），一般用来创建项目网站。简单的在github里建立一个名为<username>.github.io的repository（这同时也是github page的访问网址），就算完成了第一步

啰嗦几句

文本编辑器

Linux下推荐用vim，Windows下推荐用notepad++，这俩都有markdown插件，也不存在某些文件编辑器（说的就是你，windows notepad）在保存UTF-8编码文件时，莫名的头部BOM（vim: set nobomb）。中文用户在改动hexo框架内的文件时，建议统一使用UTF-8编码，否则容易造成现代浏览器消化不良

绑定域名

如果有自己的域名，可以在域名提供商（推荐godaddy，国内备案制度你懂的）的管理控制台（DNS management）里添加三条解析记录

Type Name Value
A @ 192.30.252.153
A @ 192.30.252.154
CNAME www username.github.io

然后在blog/source目录下创建一个名为CNAME的文件，内容为自己的域名（取决于解析设置，可以不带www）。并将站点配置文件blog/_config.yml内的url更改为自己的域名。

最后在github page的repository->settings->Custom domain里填入域名即可

除了避免不必要的js（比如上面提到的侧边栏音乐播放），一个重要的问题是默认Lato字体的加载（因为伟大的强）。为了避免这个问题，可以简单的删除主题配置文件blog/themes/next/_config.yml内相关的lato设置，并将font->global->external改为false
